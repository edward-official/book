입출력(I/O)은 주기억장치와 외부 장치==(디스크 드라이브, 터미널, 네트워크 등)== 간의 데이터를 복사하는 과정이다.
입력 작업은 I/O 장치에서 메모리로 데이터를 복사하고, 출력 작업은 메모리에서 장치로 데이터를 복사한다.

모든 프로그래밍 언어의 런타임 시스템은 I/O를 수행하기 위한 고수준 함수를 제공한다.
예를 들어 ANSI C는 `printf`와 `scanf` 같은 표준 I/O 라이브러리를 제공하며, 이는 버퍼링된 I/O를 수행한다.
C++에서는 연산자 `<<` (“put to”)와 `>>` (“get from”)으로 유사한 기능을 제공한다.

리눅스 시스템에서는 이런 고수준 I/O 함수들이 커널이 제공하는 시스템 수준의 Unix I/O 함수로 구현되어 있다.
대부분의 경우 고수준 I/O 함수로 충분하며, Unix I/O를 직접 사용할 필요는 없다.
그렇다면 왜 Unix I/O를 배워야 할까?

#### Unix I/O를 배워야 하는 이유

* Unix I/O를 이해하면 시스템 개념 전반을 이해할 수 있다.
  I/O는 시스템 동작의 핵심이다.
  예를 들어 I/O는 프로세스 생성과 실행에 중요한 역할을 하고, 반대로 프로세스 생성은 파일이 여러 프로세스 간에 어떻게 공유되는지에 영향을 준다.
  즉, I/O를 제대로 이해하려면 프로세스도 이해해야 하며 그 반대도 마찬가지다.

* 때로는 Unix I/O를 직접 써야 한다.
  표준 I/O로 접근할 수 없는 경우가 있다.
  ==예를 들어 표준 I/O는 파일 크기나 생성 시간 같은 메타데이터에 접근할 방법이 없다.==
  ==또한 네트워크 프로그래밍에서는 표준 I/O 사용이 위험할 수도 있다.==
